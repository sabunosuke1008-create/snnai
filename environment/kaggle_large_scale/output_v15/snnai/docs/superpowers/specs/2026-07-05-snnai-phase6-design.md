# SNNAI Phase6 設計書：異種ニューラルネットワーク統合

## 概要

SNNAI ロードマップの **Phase6：異種ニューラルネットワーク統合** を実装する。Phase0〜Phase5 で構築した各生物モジュールを、イベント駆動のハブと符号化変換層を介して統合する。

## 目的

- 複数の生物モジュール（線虫・ミツバチ・カラス・タコ）を1つのパイプラインで動作させる。
- モジュール間のスパイクイベントを共通フォーマットでルーティングする。
- レート符号化と時間符号化の変換により、異なる表現形式を持つモジュールを接続する。

## 統合アーキテクチャ

```
入力
 │
 ▼
[Encoding Bridge] 入力を各モジュール向けの符号化に変換
 │
 ▼
[Hub / Event Bus] スパイクイベントをモジュール間でルーティング
 │
 ┌─────────┬──────────┬─────────┬─────────┐
 ▼         ▼          ▼         ▼
c_elegans honeybee   crow     octopus
 │         │          │         │
 ▼         ▼          ▼         ▼
[Encoding Bridge] 出力を統合可能なベクトルに変換
 │
 ▼
[Final Classifier] 統合出力
```

## 構成要素

### 1. Event Spike Format

```python
{
    "timestamp": int,
    "source": str,
    "neuron_id": int,
    "strength": float,
}
```

### 2. Hub

- Pub/Sub 形式のイベントバス
- モジュールはイベントを publish/subscribe する
- 簡易実装ではリストベースの FIFO キュー

### 3. Encoding Bridge

- `spikes_to_rates`: (time, neurons) のスパイク列 -> (neurons,) の発火率
- `rates_to_spikes`: 発火率 -> ポアソンスパイク列
- モジュール間の次元・時間スケール違いを吸収

### 4. Pipeline

- 指定されたアーキテクチャに従ってモジュールを配置
- 各モジュールの出力を次のモジュールにブリッジして伝搬

## 評価方法

- 2 つ以上の実モジュールを接続し、入力から最終出力までのパイプラインが動作することを確認
- 統合後の出力が統合前の単体モジュールより高い（または同等の）性能を示す簡易タスク

## 実装方針

- snnTorch ベースのモジュールをそのまま使用
- Hub と Bridge は純粋な Python/PyTorch
- テストは `tests/test_integration.py` で2モジュール統合を確認

## 次フェーズ条件

- 少なくとも 2 つの実モジュールが統合パイプラインを通じて連携できること
- GitHub タグ `v0.7` を打つこと
