# SNNAI Phase3 設計書：ミツバチモジュール

## 概要

SNNAI ロードマップの **Phase3：ミツバチモジュール（空間認識・学習・強化学習）** を実装する。本モジュールは **Feature Learning / Sparse Memory / Reinforcement** を目的とする。

## 背景

- ミツバチの脳は約 96 万ニューロン。本モジュールではキノコ体に相当する機能を 500〜2000 ニューロン規模で模擬する。
- Phase1 の STDP と Phase2 の反射モジュールを発展させ、R-STDP による強化学習を実装する。

## 目標

1. `snnai/modules/honeybee/` に空間認識・強化学習モジュールを実装する。
2. 簡易グリッドワールドで、報酬を最大化する行動を R-STDP で学習する。
3. ランダム方策と比較して有意な学習曲線の改善を示す。

## 設計

### モジュール構造

```
snnai/modules/
└── honeybee/
    ├── __init__.py
    ├── spatial_encoding.py     # グリッドワールド状態のスパイクエンコーディング
    ├── r_stdp.py               # 報酬変調 STDP
    ├── gridworld_agent.py      # 強化学習エージェント
    └── tests/test_learning_curve.py
```

### 空間エンコーディング

- グリッドワールドの (x, y) 位置を place-cell 的なスパイク列に変換。
- 各位置に対し、近傍の place cell がガウス的に発火確率を持つ。

### R-STDP

- STDP による重み更新に報酬信号を掛け合わせる。
-  positive reward -> potentiation を促進
-  negative reward -> depression を促進

### グリッドワールド

- 簡易 2D グリッド。スタートからゴールへ移動。
- 障害物あり。ゴールに到達で正報酬、障害物で負報酬。

### 学習プロセス

1. 状態を place cell にエンコード。
2. 方策ニューラルネットワーク（SNN）が行動を選択。
3. 環境から報酬を受け取り、R-STDP で重み更新。
4. 複数エピソードで学習曲線を改善。

## 評価指標

- エピソードあたりのステップ数が減少すること。
- 累積報酬がランダム方策より有意に高いこと。

## 実装方針

- snnTorch ベース。
- 強化学習ループは PyTorch で実装。
- テストは学習曲線の改善を確認する簡易なものとする。
