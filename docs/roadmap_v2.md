# SNNAI 次期ロードマップ v2.0

> 本ロードマップは Phase7（SNNAI v1.0）完了後に作成された。v1.0 までの「基盤構築・概念実証」を踏まえ、v2.0 以降の「実データ・大規模化・実用化」方向性を示す。

---

## 1. v1.0 の振り返り

### 達成したこと

| 項目 | 状態 |
|---|---|
| Phase0〜Phase7 までの全モジュール実装 | ✅ 完了 |
| GitHub リリースタグ v0.1〜v1.0 | ✅ 完了 |
| ローカル pytest 全通過 | ✅ 完了 |
| Kaggle CPU 動作確認（Phase3） | ✅ 完了 |
| 統合パイプライン（Phase6） | ✅ 概念実証 |
| ベンチマーク基盤・ベースライン比較（Phase7） | ✅ 合成タスクで実装 |

### 残された課題

1. **実データ未対応**: 現在のベンチマークは合成データのみ。MNIST/Fashion-MNIST などの実データには未対応。
2. **モジュール間の学習が分離**: Phase6 の統合はハンドクラフトの重み固定。end-to-end 学習は未実装。
3. **規模が小さい**: 各モジュールは最小構成。大規模タスク（DVS Gesture、Atari）には不十分。
4. **GPU 活用が限定的**: Kaggle では主に CPU/P100。T4 指定の自動化が未整備。
5. **長時間学習の再開機構なし**: Kaggle のセッション制限を考慮したチェックポイント保存が不十分。
6. **Bio-NAS はサロゲート評価**: 実際の SNN モジュールを使った評価ではなく、線形代理モデルでの探索。

## 2. v2.0 の目標

### 2.1 大目標

**「v1.0 の概念を実データ・中規模タスクに拡張し、SNN 特有の強みを定量的に示す」**

### 2.2 具体的目標

1. **実データベンチマーク**: MNIST / Fashion-MNIST に対応した SNN 分類器を実装
2. **イベントデータ対応**: N-MNIST（イベントカメラ版 MNIST）の読み込み・前処理
3. **end-to-end 学習**: Phase6 の統合パイプラインを代理勾配法で end-to-end 学習可能にする
4. **大規模化**: モジュールあたり 1,000〜10,000 ニューロン規模への拡張実験
5. **GPU 最適化**: Kaggle T4 自動選択、混合精度、バッチ処理の導入
6. **チェックポイント機構**: 長時間学習のセーブ・ロード基盤
7. **改良版 Bio-NAS**: 実 SNN モジュールを使った探索（時間はかかるが、信頼性向上）

---

## 3. フェーズ計画

### Phase 8: 実データベンチマーク基盤（v1.1）

- MNIST / Fashion-MNIST 用の SNN 分類パイプライン
- snnTorch の surrogate gradient 学習を活用
- データローダーと前処理の整備
- MLP/CNN ベースラインとの比較

### Phase 9: イベントデータ対応（v1.2）

- N-MNIST データセット読み込み
- イベントデータからスパイク列への変換
- DVS Gesture への拡張を視野に入れた設計

### Phase 10: End-to-End 統合学習（v1.3）

- Phase6 の Hub/Bridge を勾配通過可能に拡張
- 複数モジュールの統合ネットワークを BPTT で学習
- ベンチマークタスクでの性能向上を確認

### Phase 11: 大規模化・GPU 最適化（v1.4）

- 各モジュールのニューロン数拡張
- Kaggle T4 / multi-GPU 対応
- 混合精度学習の導入
- チェックポイント保存・再開機構

### Phase 12: 実用タスクへの挑戦（v2.0）

- Atari 一部タイトルでの強化学習実験
- SNNAI の総合性能評価
- エネルギー効率の詳細推定
- 論文草稿向けの結果まとめ

## 5. 技術的課題と対策

| 課題 | 対策 |
|---|---|
| SNN の学習が不安定 | surrogate gradient の調整、学習率スケジュール、勾配クリッピング |
| 実データの前処理 | torchvision / tonic 等を活用した標準的な前処理パイプライン |
| Kaggle セッション制限 | エポックごとの checkpoint 保存、Kaggle Datasets への出力 |
| 大規模化によるメモリ不足 | T4 利用、バッチサイズ調整、勾配累積 |
| 統合学習の勾配消失 | skip connection、reservoir readout、層ごとの学習率調整 |

---

## 6. 評価指標

- **精度**: MNIST / Fashion-MNIST / N-MNIST での分類精度
- **パラメータ効率**: 同精度達成に必要なパラメータ数
- **推論速度**: 1 サンプルあたりの推論時間
- **エネルギー推定**: スパイク数に基づく消費電力推定
- **Atari スコア**: ベースライン（DQN 等）との比較

---

## 7. リスク

- **Atari 到達は困難かもしれない**: 必要に応じて MiniGrid や PyBullet 等の中間目標を設定
- **SNN ライブラリの制約**: snnTorch 以外（Norse, Spyx）への切り替え検討を並行して行う
- **Kaggle の GPU 制限**: ローカル GPU やクラウド環境への移行も検討

---

## 8. 結論

v2.0 の最終目標は、**v1.0 で示したモジュラーSNNアーキテクチャを実データと中規模タスクに拡張し、SNN の省電力・モジュール性の強みを定量的に示す**ことである。Phase8〜Phase12 を順番に進め、各フェーズで GitHub タグを打ち、段階的に成果を蓄積していく。
