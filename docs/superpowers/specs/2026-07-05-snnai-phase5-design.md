# SNNAI Phase5 設計書：タコ分散モジュール

## 概要

SNNAI ロードマップの **Phase5：タコ分散モジュール（独立並列処理・多モーダル統合）** を実装する。本モジュールは **Independent Parallel Processing / Multi-sensory Integration** を目的とする。

## 背景

- タコは多数の独立した腕を持ち、各腕に局所的な「ミニ脳」がある。
- 本モジュールでは複数の独立 SNN サブモジュールを並列に動作させ、上位モジュールで統合する。

## 目標

1. `snnai/modules/octopus/` に分散並列モジュールを実装する。
2. 複数の入力チャネルを独立したサブネットワークで並列処理し、統合層で判定する。
3. 少なくとも 2 つの独立サブモジュールが並列動作し、結果が統合されることを示す。

## 設計

### モジュール構造

```
snnai/modules/
└── octopus/
    ├── __init__.py
    ├── distributed_module.py
    └── tests/test_parallel_integration.py
```

### 分散並列モジュール

- **入力**: 複数のセンサチャネル（例：視覚、触覚）
- **サブモジュール**: 各センサチャネル専用の小さな LIF ネットワーク
- **統合層**: 各サブモジュールの出力を集約し、最終判定を出力

### タスク例

- 両方のチャネルに強い刺激があれば「衝突警告」
- 片方だけなら「注意」
- どちらも弱ければ「安全」

## 評価指標

- 並列実行されたサブモジュールの出力が統合層で正しく分類されること
- 処理時間が直列実行より短いこと（CPU 並列化の有無は問わない）

## 実装方針

- Python の `concurrent.futures.ThreadPoolExecutor` または単純なリスト内包で並列実行を模擬
- 各サブモジュールは `snnai/core/neurons/lif.py` を利用可能
- テストでは統合層の出力形状と分類正解を確認

## 次フェーズ条件

- GitHub タグ `v0.5` を打つこと
