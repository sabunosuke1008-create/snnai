# SNNAI Phase X 設計書：Bio-NAS（生物制約付きアーキテクチャ探索）

## 概要

SNNAI ロードマップの **Phase X：Bio-NAS** を実装する。Phase0〜Phase5 までに作成した生物モジュール（線虫・ミツバチ・カラス・タコ）の接続トポロジーを、生物学的制約付きで進化的に探索する。

## 目的

- 固定された「線虫→蜂→鳥→タコ」直列構造が最適とは限らないことを実証する。
- 小規模な代理タスク上で複数の接続パターンをスクリーニングし、有望な構造を選出する。
- Phase6 の統合構造決定に探索結果を反映できるようにする。

## 探索空間

### モジュール種別

- `c_elegans`: 反射・入力近傍
- `honeybee`: 特徴学習・強化学習
- `crow`: 作業記憶
- `octopus`: 分散並列処理

### 接続テンプレート

1. `serial`: 入力 → 線虫 → 蜂 → 鳥 → タコ → 出力
2. `hub`: 全モジュールが中央ハブに接続
3. `parallel`: 線虫を入力側に固定し、蜂/鳥/タコを並列に配置して統合

### 生物制約（ハード制約）

- `c_elegans` は入力から最短距離（1 ステップ）に配置
- 最終出力の直前には必ず 1 つの統合モジュールを配置
- グラフは有向非巡回（DAG）であること

## 評価方法

合成多モーダル分類タスクを使用：
- 入力：2 チャネルの時系列データ
- 各アーキテクチャは、サブモジュールを接続して出力を生成
- 目標：出力の分類精度を最大化

評価は軽量かつ確定的に行い、進化の世代ごとに多数候補を評価できるようにする。

## 実装方針

- `search_space.py`: アーキテクチャグラフの定義と制約チェック
- `evaluator.py`: 合成タスクでの性能評価
- `evolution_search.py`: 進化的探索（突然変異＋選択）
- `tests/test_bio_nas.py`: 探索空間の有効性と探索結果の妥当性を確認
